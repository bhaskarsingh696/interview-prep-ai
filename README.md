# 🎯 Smart Interview Platform

An AI-powered platform for mock interviews, DSA practice, and career development with real-time video sessions, resume analysis, and peer interviews.

![Status](https://img.shields.io/badge/Status-Beta-yellow) ![License](https://img.shields.io/badge/License-MIT-green) ![Node](https://img.shields.io/badge/Node-18+-blue) ![React](https://img.shields.io/badge/React-19-blue)

---

## ✨ Features

### 🤖 **AI Mock Interviews**

- Questions generated by Groq's Llama 3.3 AI
- Real-time evaluation and feedback
- Topics: DSA, OS, DBMS, CN, HR
- Performance tracking

### 💻 **Code Editor**

- Multi-language support (C++, Python, JavaScript)
- Monaco Editor with syntax highlighting
- Real-time code execution via Piston API
- Problem submission tracking

### 👥 **Peer Interviews**

- Real-time video/audio (WebRTC)
- Live code collaboration via Socket.io
- Peer feedback system
- Room management

### 📊 **Analytics Dashboard**

- Problem-solving statistics
- Weak topic identification
- Performance trends
- Mock interview history

### 📄 **Resume Analyzer**

- AI-powered resume analysis
- Skill extraction
- Job role suggestions
- Learning roadmap generation
- PDF parsing

### 🔐 **Authentication**

- JWT-based auth
- Secure password hashing (bcryptjs)
- Token refresh mechanism
- Role-based access (student/interviewer)

## 🚀 Quick Start

### 1️⃣ Clone Repository

```bash
git clone https://github.com/yourusername/smart-interview-platform.git
cd smart-interview-platform
```

### 2️⃣ Backend Setup

```bash
cd backend

# Install dependencies
npm install

# Create .env file from example
cp .env.example .env

# Edit .env with your credentials
# MONGO_URI: MongoDB connection string
# JWT_SECRET: Random 32-char string
# GROQ_API_KEY: From console.groq.com
nano .env

# Start development server
npm run dev
```

✅ Backend running on `http://localhost:5000`

### 3️⃣ Frontend Setup

```bash
cd frontend

# Install dependencies
npm install

# Create .env file
cp .env.example .env

# Start development server
npm start
```

✅ Frontend running on `http://localhost:3001`

## 📚 API Documentation

### Authentication Endpoints

```
POST /api/auth/register      - Register new user
POST /api/auth/login         - Login user
GET  /api/auth/me            - Get current user
POST /api/auth/logout        - Logout & blacklist token
```

### Code Practice

```
GET  /api/code/problems              - Fetch problems (paginated)
GET  /api/code/problems/:id          - Get problem details
POST /api/code/run                   - Execute code
POST /api/code/submit                - Submit solution
GET  /api/code/submissions/me        - Get user submissions
```

### AI Interviews

```
POST /api/ai/questions               - Generate interview questions
POST /api/ai/evaluate                - Evaluate answer
POST /api/ai/interview               - Save interview session
GET  /api/ai/interviews/me           - Get interview history
```

### Resume Analysis

```
POST /api/resume/upload              - Upload & analyze resume
GET  /api/resume/my                  - Get user's resumes
```

### Analytics

```
GET  /api/analytics                  - Get dashboard stats
GET  /api/analytics/weak-topics      - Identify weak areas
```

### Peer Interviews

```
POST /api/peer/create-room           - Create interview room
GET  /api/peer/available-rooms       - List available rooms
POST /api/peer/join/:roomId          - Join room
POST /api/peer/complete/:roomId      - Complete & submit feedback
```

## 📊 Database Schema

### User

```javascript
{
  name: String,
  email: String (unique),
  password: String (hashed),
  role: 'student' | 'interviewer',
  avatar: String,
  createdAt: Date
}
```

### Problem

```javascript
{
  title: String,
  description: String,
  difficulty: 'Easy' | 'Medium' | 'Hard',
  topic: 'DSA' | 'OS' | 'DBMS' | 'CN' | 'HR',
  tags: [String],
  examples: [{input, output, explanation}],
  starterCode: {cpp, python, javascript},
  solution: String,
  createdAt: Date
}
```

### Submission

```javascript
{
  user: ObjectId,
  problem: ObjectId,
  code: String,
  language: 'cpp' | 'python' | 'javascript',
  status: 'Accepted' | 'Wrong Answer' | 'Runtime Error' | 'TLE',
  timeTaken: Number,
  createdAt: Date
}
```

## 🚀 Deployment

### Option 1: **Frontend on Vercel** (Recommended)

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy
cd frontend
vercel --prod

# Add environment variables in Vercel dashboard
REACT_APP_API_URL=https://your-api.com/api
```

### Option 2: **Backend on Render**

1. Push code to GitHub
2. Sign up at https://render.com
3. Create new Web Service from GitHub repo
4. Add environment variables
5. Deploy

### Option 3: **Docker Deployment**

```bash
# Build images
docker-compose build

# Run containers
docker-compose up -d

# Access at http://localhost
```
